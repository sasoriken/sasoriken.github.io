# 00. ポートフォリオ コンセプト文書

このドキュメントは sasoriken のポートフォリオサイト全体の設計指針を定めたものです。後工程の AI エージェント（プロジェクト素材ファイル生成、サイト構造設計、HTML 生成）は、すべてこのドキュメントを最上位の参照仕様として扱ってください。各段階の文書 (`01_*.md`, `02_*.md`, ...) よりも本書が優先されます。

---

## 1. 基本軸

このサイトの中心は **「人物」ではなく「作品」** です。

訪問者がトップページに来てまず目にすべきは "sasoriken が作っているもの一覧" であり、自己紹介や経歴は補助的な位置に置きます。「誰が作っているか」より「何が作られているか」を先に見せる。プロフィールページは存在してよいが、主従関係としては作品が主、プロフィールが従。

## 2. 主目的

第一目的は **自作ツールの認知・利用拡大** です。

訪問者の主要動線として想定するのは:

1. トップで作品一覧を眺める
2. 気になった作品の個別情報を読む
3. 配布先（BOOTH / itch.io / GitHub）へ離脱して購入・ダウンロードする

副次的な目的として、就職・転職活動時の自己説明にも転用できること。ただし副次的なので、サイト設計が就活向けに引きずられないよう注意する。

## 3. 想定読者

幅広く誰でも。創作者・開発者・エンドユーザーのいずれかに閉じない。

ただし日英併記とするため、海外からの訪問者にとっても基本的な情報が理解可能な作りにすること。

## 4. トーン

個性的・遊び心ありの方向。具体的な指針:

- 装飾過多にしない。ただし無味乾燥にもしない
- 配色・見出し・マイクロインタラクションで "らしさ" を演出する
- 文体は「淡々と事実」より「少し主観や癖がある」方を選ぶ
- ふざけすぎず、技術者・販売者としての信頼性は損なわない
- 「個性を出す」と「購入・利用判断の邪魔にならない」を両立する

遊び心を強く出してよい箇所:

- トップページのヒーロー部
- フッター
- 404 / エラーページ
- About / プロフィール部

逆に遊び心を抑えるべき箇所:

- 各プロジェクトの紹介本文（読みやすさ最優先）
- 配布リンクや連絡先の周辺（信頼性最優先）

## 5. 言語

**日英併記** を初期版から採用する。

実装方針:

- 各ページに言語切り替えボタン（JP / EN）を持たせる
- 初期表示は訪問者のブラウザ言語設定に応じて自動切り替え。フォールバックは日本語
- 各プロジェクト素材ファイルには日英両方の `one_liner` と `概要` を持たせる
- メタタイトル・OGP も両言語版を用意する
- URL 構造は `/` (JP デフォルト) と `/en/` (EN) で分ける案を第4段階で検討

## 6. アイデンティティ

- 表示名: **sasoriken**（ローマ字表記で統一）
- 日本語ページにおいても「sasoriken」とローマ字表記する
- 公開範囲: **ハンドル名のみ**
  - 本名・年齢・地域・職業・経歴・顔写真などの個人情報は一切出さない
  - 「中の人」を匂わせる情報も出さない
- アイコン画像: 要・本人記入（後から決定）

## 7. 連絡手段

サイトに掲載する連絡手段は以下の 5 つ。それぞれの URL は要・本人記入。

| 種別 | 用途 | URL |
|---|---|---|
| X (Twitter) | 一般的な接点・近況発信 | 要・本人記入 |
| GitHub | コード関連の窓口・公開リポジトリ | 要・本人記入 |
| メールアドレス | フォーマルな連絡 | 要・本人記入（画像化または難読化推奨） |
| BOOTH | 有料配布作品の購入 | 要・本人記入 |
| itch.io | 有料・無料配布作品の購入／DL | 要・本人記入 |

連絡先セクションはトップページからすぐ辿れる場所と、フッター両方に配置する。

## 8. 載せるもの／載せないもの

**載せるもの:**

- 開発・配布しているツール（有料・無料問わず）
- 試作・実験プロジェクトのうち、人に見せられる状態のもの
- BOOTH / itch.io の配布ストアへの導線
- GitHub プロフィール・主要リポジトリへのリンク

**載せないもの:**

- 連載中の小説（このポートフォリオの範囲外）
- 個人情報全般（本名・経歴・写真・所属など）
- 未完成・公開判断未了のプロジェクト（隠す）

## 9. 配布形態と作品の傾向

- 配布スタイル: 有料と無料がほぼ半々
- ジャンル: 多岐にわたる（特定領域に閉じない）

この特性から、サイト設計上の含意は以下のとおり:

- 「ジャンル別タブ」より「カテゴリ／ステータス／配布形態でフィルタできる横断的な作品一覧」の方が向いている
- 「有料／無料」「現役配布中／アーカイブ」を視覚的に区別できるようにする
- 価格表記の有無で印象が変わるため、ストアへのリンクボタンは統一されたデザインで提示する

具体的なフィルタ仕様・ソート仕様は第4段階で確定する。

## 10. 代表作 (featured) の扱い

第2段階のプロジェクト棚卸し完了後に、`featured: true` とする項目を決定する。

決定までの暫定方針:

- 各素材ファイルの `featured` 初期値は `false`
- 棚卸し終了時に 3〜5 件を選定して `true` に変更する
- featured 作品はトップページで大きく取り扱う

## 11. 後工程への申し送り（重要）

第2段階以降の作業を行う AI および本人は、必ず以下を遵守すること。

1. **各プロジェクト素材ファイル (`projects/*.md`) は日英両方のテキストを含むこと。** `one_liner` と `概要` セクションは必ず両言語版を用意する。
2. **HTML 生成時は JP / EN の両言語版を出力すること。** 片方のみの出力は受け入れない。
3. **遊び心を出す箇所**は本書第4節で定めた範囲に集中させ、プロジェクト紹介本文は読みやすさを優先すること。
4. **連絡先情報のうち URL がまだ確定していないものはプレースホルダ** `（要・本人記入）` を残し、絶対に創作しないこと。
5. **個人情報を匂わせる文言を生成しないこと。** プロフィール文の自動生成時、年齢・所属・経歴を推測して書かない。

## 12. 未確定事項のリスト

このコンセプト段階で確定していない、または保留にした項目:

- アイコン画像の有無・デザイン
- 各種 URL（X / GitHub / Email / BOOTH / itch.io）
- 代表作の選定（第2段階完了後）
- 配色とフォントの具体（第4段階で決定）
- ホスティング先（GitHub Pages を第一候補とするが第5段階で最終決定）
- 静的サイトジェネレータの採用可否（同上）
